Why Iva

OpenClaw, Hermes, nanobot — there are a lot of good agents out there. So why another one? Because every one of them hands you the same pile of choices: which model, which memory, which search, how to wire it together. Too much choice is the real problem. So I made the choices: I test agents, models, stacks and harnesses all the time, keep what's actually best, and fold it into Iva — the best of every agent in one. Open code on open models (DeepSeek, Kimi, GLM) — you pick the model by name, your key, no markup. The Linux Mint of AI agents: one command, and it just works. Lots of agents out there; this one's mine — now it's yours too.

What it can do

  • 🎙️ Voice & video — transcribes voice notes and video circles, understands many languages.
  • 🧠 Long-term memory — a tree of plain files, no heavy database.
  • 🔎 Fast recall — finds the right note in seconds.
  • ⏰ On a schedule — day or week digests, recurring jobs, inbox summaries, on time.
  • 🔔 Reminders — tell it what and when, it won't let you forget.
  • 🤖 Your choice of model — DeepSeek, Kimi, GLM and other open models.
  • 🌐 Does more — web search, opens pages, drives a browser, connects to MCP servers.
  • 🔒 Private — the code and the memory stay on your server, not in someone's cloud.

Memory — the part that compounds

Iva's memory is shaped like a tree (and iva means willow). It's "low-context memory" by design: never the whole history in the model, just a tiny always-on core plus whatever a task needs, found by a literal search over plain files.

🪵 TRUNK — year + cards on people, projects, decisions ╱ ╲ 🌿 BRANCHES — monthly summaries, built from weeks, built from days ╱ ╲ 🍃 LEAVES — the full, word-for-word transcript of each day

Every night Iva summarizes the leaves and folds them up the branches — a hierarchical summary DAG that compacts older days while keeping a pointer to every original. It's the part I've worked on longest (agent-second-brain → autograph → Iva), built on the LCM: Lossless Context Management paper plus a lot of practice — one of the best memory designs you'll find in a personal agent today. Zero infrastructure, fully inspectable, git-diffable — no vector DB to run, sync or pay for.

Providers & cost

Free and open-source. You pay only for a small always-on server (a $4 droplet — 1 CPU, 512 MB, 10 GB) and one model subscription — OpenCode Zen Go (~$5/mo) or Ollama Cloud (~$20/mo), both OpenAI-compatible, your own key. About $9 a month all in. Voice runs on Deepgram (free starter credit).
